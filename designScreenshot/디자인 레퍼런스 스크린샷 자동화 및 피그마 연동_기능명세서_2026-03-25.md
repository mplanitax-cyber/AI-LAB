## 1. 프로젝트 개요
---
### 카테고리
> 생산성 / 업무

### 한 줄 정의
디자인 레퍼런스 스크린샷 자동화 및 피그마 연동

### 배경
UI/UX 디자이너들은 디자인 프로젝트를 진행할 때 다양한 레퍼런스 화면을 수집하고 분석합니다. 기존에는 웹 확장 프로그램이나 개별 캡처 도구를 사용하여 웹, 태블릿, 모바일 화면을 각각 캡처한 후, 이를 피그마로 가져와 수동으로 정리하는 번거로운 과정을 거쳤습니다. 이러한 수작업은 많은 시간과 노력을 요구하며, 레퍼런스가 업데이트될 경우 다시 캡처해야 하는 비효율성을 초래합니다.

### 사용자 문제
디자이너들은 다음과 같은 문제에 직면해 있습니다. 첫째, 여러 URL의 웹/태블릿/모바일 화면을 각각 수동으로 캡처하고 피그마로 가져오는 과정이 매우 비효율적입니다. 둘째, 캡처된 스크린샷은 단순 이미지 파일이어서 피그마 내에서 편집하거나 실시간으로 업데이트 내용을 반영하기 어렵습니다. 셋째, 수많은 레퍼런스를 체계적으로 관리하고 디자인 시스템에 맞춰 정리하는 데 어려움을 겪습니다.

### 해결 방식
사용자가 최대 10개(유료 플랜 시 20개 이상)의 URL을 한 번에 입력하면, 해당 웹사이트의 웹, 태블릿, 모바일 세 가지 버전의 스크린샷을 자동으로 캡처하여 '살아있는 레이어(Live Layer)' 형태로 피그마로 내보낼 수 있는 웹 서비스를 제공합니다. 이 서비스는 디자이너의 수동 작업을 최소화하고, 피그마 내에서 레퍼런스 화면을 효율적으로 관리할 수 있도록 돕습니다. 초기에는 웹 서비스로 제공되며, 향후 피그마 플러그인 연동을 통해 더욱 원활한 워크플로우를 구축할 예정입니다.

### 차별점
경쟁 서비스 대비 다음과 같은 차별점을 가집니다: 1) 다중 URL 및 다중 기기 스크린샷 동시 캡처 기능으로 압도적인 효율성을 제공합니다. 2) 캡처된 스크린샷을 단순 이미지가 아닌, 피그마에서 활용 가능한 '라이브 레이어' 형태로 내보내어 디자인 작업에 즉시 통합될 수 있도록 합니다. 3) 구독 모델을 통해 사용자 요구에 맞는 유연한 사용 범위를 제공하며, 향후 예정된 스크린샷 자동 업데이트 및 디자인 시스템 정리 기능은 레퍼런스 관리의 패러다임을 바꿀 것입니다.

### 타겟 사용자
일상적으로 웹사이트, 태블릿, 모바일 앱 디자인 레퍼런스를 수집하고 분석하며, 피그마를 주요 디자인 도구로 사용하는 UI/UX 디자이너. 특히, 여러 웹페이지를 동시에 다양한 기기 화면으로 캡처하고 이를 피그마에서 즉시 활용하고자 하는 디자이너가 핵심 타겟입니다.

### 사용 시나리오
UI/UX 디자이너가 새로운 프로젝트를 시작합니다. 경쟁사 서비스 벤치마킹을 위해 관련 웹사이트 5개의 URL을 본 서비스에 입력합니다. 서비스는 각 URL에 대해 웹, 태블릿, 모바일 버전의 스크린샷을 자동으로 생성하고, 디자이너는 이를 클릭 한 번으로 피그마 작업 파일에 '라이브 레이어' 형태로 내보냅니다. 피그마에서 이 레이어들을 그룹화하고 필요한 부분을 수정하며 디자인 작업을 진행합니다. 유료 플랜을 구독한 디자이너는 20개 이상의 URL을 등록하고, 주간 단위로 스크린샷이 자동으로 업데이트되도록 설정하여 항상 최신 레퍼런스를 유지합니다.

### 제품 목표
UI/UX 디자이너가 수동으로 웹 화면 레퍼런스를 수집하고 정리하는 반복적인 작업을 자동화하여 시간과 노력을 절감하고, 다양한 기기(웹, 태블릿, 모바일)의 스크린샷을 피그마(Figma) 라이브 레이어로 효율적으로 관리할 수 있도록 지원하여 디자인 워크플로우의 생산성을 극대화합니다.

### 핵심 KPI
월간 활성 사용자 수(MAU), 유료 구독 전환율, 사용자당 평균 스크린샷 캡처 횟수, 피그마 내보내기 기능 사용률, 사용자당 피그마 파일 연동 수, 기능별(다중 URL, 다중 기기) 사용 빈도.

### 리스크/이슈
안정적인 스크린샷 캡처 기술 확보(특히 동적인 웹 페이지), 피그마 API 연동의 기술적 난이도 및 안정성 유지, 기존 피그마 플러그인과의 차별점 부각 및 경쟁 심화, 사용자 데이터 프라이버시 및 보안 문제, 구독 모델 수익성 확보의 어려움.

### 사용자 역할
> 일반 사용자, 유료 구독자

### 디바이스 (기기)
> Web, Figma Plugin, mo, tablet

## 1. 다중 URL 입력 및 스크린샷 캡처
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

사용자가 최대 10개의 URL을 동시에 입력하여 각 URL에 대한 스크린샷을 한 번에 캡처할 수 있어야 합니다. 유료 플랜 구독 시 20개 이상의 URL 입력이 가능해야 합니다.

**수용 기준:**
```
1. ○ 사용자는 입력 필드에 여러 URL을 한 줄에 하나씩 입력할 수 있다.
2. ○ 각 URL에 대해 스크린샷 캡처가 성공적으로 이루어져야 한다.
3. ○ 무료 사용자는 최대 10개의 URL, 유료 사용자는 20개 이상의 URL을 입력할 수 있다.
4. ○ 캡처 실패 시 사용자에게 명확한 실패 원인과 함께 재시도 옵션을 제공해야 한다.
```


### 1.1 URL 입력 폼 제공

사용자가 여러 개의 URL을 입력할 수 있는 텍스트 입력 폼을 제공합니다. 무료 사용자는 최대 10개, 유료 사용자는 20개 이상의 URL을 입력할 수 있도록 제한을 적용합니다.

사용자가 여러 개의 URL을 한 줄에 하나씩 입력할 수 있는 텍스트 영역(textarea) 또는 태그 입력 필드를 제공합니다. 입력 필드는 최소 500px의 높이와 충분한 너비를 가져야 하며, 자동 완성 및 필드 제한 안내 메시지가 표시되어야 합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web


#### 1.1.2 URL 입력 제한 및 카운터

무료 사용자는 최대 10개, 유료 사용자는 20개 이상의 URL을 입력할 수 있도록 제한합니다. 현재 입력된 URL 개수를 실시간으로 카운팅하여 '입력된 URL: 5/10'과 같은 형식으로 표시합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web

사용자가 입력한 모든 URL을 한 번에 지울 수 있는 '초기화' 또는 '모두 지우기' 버튼을 제공합니다. 클릭 시 확인 다이얼로그를 표시하여 실수로 인한 삭제를 방지합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web


### 1.2 다중 URL 유효성 검사
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web

입력된 URL들의 형식을 검증하고, 유효하지 않은 URL에 대해 사용자에게 즉시 피드백을 제공합니다. 중복 URL은 자동으로 필터링합니다.


#### 1.2.1 URL 형식 검증

입력된 각 URL이 유효한 웹 주소 형식(http://, https://)을 가지고 있는지 검증합니다. 유효하지 않은 형식의 URL은 빨간색으로 표시하고 오류 메시지를 함께 표시합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web


#### 1.2.2 중복 URL 필터링

사용자가 입력한 URL 목록에서 중복된 주소를 자동으로 감지하고 제거합니다. 중복 감지 시 사용자에게 '3개의 중복 URL이 제거되었습니다'라는 알림 메시지를 표시합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web


#### 1.2.3 리얼타임 유효성 검사 피드백

사용자가 URL을 입력하거나 수정할 때마다 실시간으로 유효성 검사를 수행하고, 각 URL 옆에 검증 상태 아이콘(✓ 또는 ✗)을 표시합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web


### 1.3 배치 스크린샷 캡처 시작
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web

사용자가 입력한 URL 목록에 대해 일괄 스크린샷 캡처 프로세스를 시작할 수 있는 버튼과 진행 상황 표시 기능을 제공합니다.


#### 1.3.1 배치 캡처 시작 버튼

유효한 URL이 모두 입력된 후 사용자가 클릭하여 배치 스크린샷 캡처 프로세스를 시작할 수 있는 '캡처 시작' 또는 '스크린샷 생성' 버튼을 제공합니다. 버튼은 유효한 URL이 없을 때는 비활성화됩니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web


#### 1.3.2 캡처 진행 상황 표시

스크린샷 캡처가 진행 중일 때 전체 진행률을 백분율(예: 45%)로 표시하고, 현재 처리 중인 URL(예: '처리 중: www.example.com')을 함께 표시합니다. 프로그레스 바를 사용하여 시각적으로 진행 상황을 보여줍니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web


#### 1.3.3 캡처 완료 화면 전환

모든 URL에 대한 스크린샷 캡처가 완료되면 결과 화면으로 자동으로 전환합니다. 결과 화면에는 캡처된 스크린샷의 썸네일, URL, 기기별 버전이 표시됩니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web


### 1.4 캡처 실패 처리 및 재시도
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web

스크린샷 캡처 실패 시 실패 원인(예: 네트워크 오류, 타임아웃, 접근 제한)을 사용자에게 명확하게 표시하고, 개별 URL에 대한 재시도 옵션을 제공합니다.


#### 1.4.1 실패 원인 분류 및 표시

스크린샷 캡처 실패 시 구체적인 원인(예: '네트워크 연결 실패', '페이지 로드 타임아웃', '서버 접근 제한', '보안 정책 위반')을 분류하여 사용자에게 명확하게 표시합니다. 각 실패 항목 옆에 실패 원인을 아이콘과 텍스트로 표시합니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web


#### 1.4.2 개별 URL 재시도 기능

실패한 각 URL에 대해 '재시도' 버튼을 제공하여 사용자가 해당 URL만 다시 캡처할 수 있도록 합니다. 재시도 시 해당 URL에 대해서만 캡처 프로세스가 다시 실행됩니다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web


#### 1.4.3 일괄 재시도 기능

실패한 모든 URL을 한 번에 재시도할 수 있는 '모두 재시도' 버튼을 제공합니다. 클릭 시 이전 배치 캡처 중 실패했던 모든 URL에 대해 다시 캡처를 시작합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 일반 사용자, 유료 구독자 | **기기**: Web

## 2. 다중 기기(웹, 태블릿, 모바일) 스크린샷 생성
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

입력된 각 URL에 대해 웹(데스크톱), 태블릿, 모바일 세 가지 기기 환경에서의 화면 스크린샷을 자동으로 생성해야 합니다.

**수용 기준:**
```
1. ○ 각 URL에 대해 웹, 태블릿, 모바일 해상도(예: 1920x1080, 768x1024, 375x812)로 스크린샷이 생성된다.
2. ○ 생성된 스크린샷은 각 기기별로 명확하게 구분되어 표시되어야 한다.
3. ○ 스크롤이 필요한 긴 페이지도 전체 화면이 캡처되어야 한다.
```

## 5. 스크린샷 자동 업데이트 및 디자인 시스템 정리 기능
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전

사용자가 설정한 주기에 따라 등록된 URL의 스크린샷을 자동으로 업데이트하고, 캡처된 레퍼런스를 디자인 시스템 항목별로 정리할 수 있는 기능을 제공합니다.

**수용 기준:**
```
1. ○ 사용자는 특정 URL에 대해 스크린샷 자동 업데이트 주기를 설정할 수 있다 (예: 일간, 주간, 월간).
2. ○ 자동 업데이트 시 변경된 스크린샷을 사용자에게 알림으로 제공한다.
3. ○ 사용자는 캡처된 스크린샷에 태그를 지정하거나 카테고리별로 분류하여 디자인 시스템 항목에 맞춰 정리할 수 있다.
4. ○ 정리된 스크린샷을 피그마 내에서 구조화된 형태로 불러올 수 있다.
```

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 유료 구독자 | **기기**: Web
